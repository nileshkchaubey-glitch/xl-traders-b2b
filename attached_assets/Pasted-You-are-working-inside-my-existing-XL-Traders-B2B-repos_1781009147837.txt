PHASE 1 — DATABASE MIGRATIONS (generate SQL files only)
═══════════════════════════════════════════

Create these migration files:

migration-02-sku-barcode.sql:
- ALTER products ADD COLUMN sku TEXT
- ALTER products ADD COLUMN barcode TEXT  
- ALTER products ADD COLUMN moq INTEGER DEFAULT 1
- Backfill sku for ALL existing products: auto-generate as 
  {first-3-letters-of-category-name}-{4-digit-sequence} e.g. BAL-0001, 
  BAC-0001. Category name comes from joining categories table.
- After backfill: CREATE UNIQUE INDEX ON products(sku)
- Add index on barcode (not unique — same barcode can exist for variants)

migration-03-orders.sql:
- CREATE TABLE orders (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    customer_name TEXT,
    phone TEXT,
    status TEXT DEFAULT 'new' CHECK (status IN ('new','confirmed','processing','delivered','cancelled')),
    total_amount NUMERIC,
    item_count INTEGER,
    notes TEXT,
    source TEXT DEFAULT 'cart'
  )
- CREATE TABLE order_items (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    order_id UUID REFERENCES orders(id) ON DELETE CASCADE,
    product_id UUID REFERENCES products(id),
    sku TEXT,
    product_name TEXT,
    quantity INTEGER NOT NULL,
    unit_price NUMERIC,
    unit_of_measure TEXT,
    subtotal NUMERIC
  )
- RLS: anyone can INSERT orders + order_items (website visitors). 
  Only authenticated can SELECT/UPDATE/DELETE.
- Index on orders(created_at DESC)

migration-04-import-logs.sql:
- CREATE TABLE import_logs (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    source TEXT,
    rows_total INTEGER,
    inserted INTEGER DEFAULT 0,
    updated INTEGER DEFAULT 0,
    skipped INTEGER DEFAULT 0,
    errors JSONB DEFAULT '[]'
  )
- RLS: only authenticated can read/write.

Do NOT create audit_logs, product_attributes, attribute_values, or 
product_tags tables. Out of scope.

PHASE 3 — CART SYSTEM (the big one)
═══════════════════════════════════════════

This is a B2B wholesale cart. NO payment gateway. Orders are placed via 
WhatsApp. The flow: browse → add to cart → review cart → place order 
via WhatsApp (saves to DB + opens WhatsApp with formatted message).

3a. Cart Store (Zustand):
- Create client/src/stores/cartStore.ts using Zustand with localStorage 
  persistence (persist middleware)
- State: items[] (productId, sku, name, price, quantity, unit, imageUrl), 
  customer info (name, phone)
- Actions: addItem, removeItem, updateQuantity, clearCart, getTotal, 
  getItemCount
- MOQ soft validation: if quantity < product.moq, show warning but allow

3b. Cart UI Components:
- AddToCartButton component: quantity selector + "Add to Cart" button. 
  Show on ProductCard and ProductDetail. Keep existing "Enquire on 
  WhatsApp" button too (don't remove it).
- CartDrawer/CartPage: list items, edit quantities, remove items, show 
  subtotals + total, customer name + phone input
- CartIcon in header: show badge with item count
- "Place Order via WhatsApp" button that:
  1. Validates cart not empty, name + phone filled
  2. POSTs to orders table (order + order_items)
  3. Then opens WhatsApp with formatted message:
     "🛒 New Order from XL Traders\n
      Customer: {name}\nPhone: {phone}\n
      ──────────\n
      {for each item: qty x product_name — ₹subtotal}\n
      ──────────\n
      Total: ₹{total}\n
      Items: {count}"
  4. Clears cart after successful order
